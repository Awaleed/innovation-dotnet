# Innovation Platform — Deployment Guide

## Prerequisites

- **Docker Compose:** Docker Engine 24+ with Compose V2
- **Kubernetes:** kubectl (optional, for K8s deployment)
- **Python 3.9+** with PyYAML (`pip install pyyaml`) — for the publish script

## Building

```bash
# Build and push container image + generate deployment artifacts
python publish.py --target all --image-tag v1.0.0

# Build for Docker Compose only (skip push for local testing)
python publish.py --target docker --image-tag v1.0.0 --skip-push

# Build for Kubernetes only
python publish.py --target kubernetes --image-tag v1.0.0 --skip-push
```

Output goes to `dist/docker-compose/` and `dist/kubernetes/`.

## Option A — Docker Compose (Recommended)

Best for single-server on-prem deployments.

### Quick Start

```bash
cd dist/docker-compose

# 1. Edit .env — set DOMAIN and AUTH_DOMAIN
#    Passwords are auto-generated by publish.py
nano .env

# 2. Add hosts entries (for local testing with .test domains)
#    echo "127.0.0.1 innovation.test auth.innovation.test" >> /etc/hosts

# 3. Start all services
docker compose up -d
```

### Architecture

```
Browser → Caddy (TLS) → Web App (:8080)
                       → Keycloak (:8080)

Internal network (not exposed):
  PostgreSQL (:5432) — app data + Keycloak data
  Redis (:6379)      — cache + session revocation
  OpenLDAP (:389)    — user directory
```

### Entry Points

| Service | URL | Notes |
|---------|-----|-------|
| Web App | `https://{DOMAIN}` | Main application |
| Keycloak | `https://{AUTH_DOMAIN}` | Auth + admin console |
| Dashboard | `http://localhost:18888` | Aspire observability (localhost only) |

### TLS Configuration

Caddy handles TLS automatically. Three modes:

| Mode | Caddyfile Config | When to Use |
|------|-----------------|-------------|
| **Auto** (Let's Encrypt) | Default | Public-facing with valid domain |
| **Custom certs** | `tls /etc/caddy/certs/cert.pem /etc/caddy/certs/key.pem` | Corporate CA |
| **Self-signed** | `tls internal` | Local testing / air-gapped |

For self-signed (`.test` domains), edit `Caddyfile` and uncomment `tls internal`.

### Updating

```bash
# Pull new web image
docker compose pull web

# Restart with new image
docker compose up -d web
```

### Backups

```bash
# Manual backup
./backup.sh

# Automated daily backup (add to cron)
0 2 * * * cd /path/to/docker-compose && ./backup.sh
```

Backups are saved to `./backups/` with 30-day retention.

## Option B — Kubernetes

Best for clusters, HA, or enterprise environments.

```bash
cd dist/kubernetes

# 1. Replace domain placeholders in manifests
sed -i 's/DOMAIN_PLACEHOLDER/innovation.test/g; s/AUTH_DOMAIN_PLACEHOLDER/auth.innovation.test/g' *.yaml

# 2. Deploy
bash deploy.sh
```

Requires: nginx Ingress controller + cert-manager (or manual TLS secret).

## Option C — Windows Native

Run the web app as a native Windows EXE with Docker only for infrastructure.

```powershell
# 1. Start infrastructure
cd dist\docker-compose
docker compose up -d postgres redis keycloak openldap

# 2. Run the web app
cd ..\windows
$env:ConnectionStrings__innovationdb = "Host=localhost;Port=5432;Database=innovationdb;Username=postgres;Password=YOUR_PASSWORD"
$env:ConnectionStrings__redis = "localhost:6379,password=YOUR_PASSWORD"
$env:ConnectionStrings__keycloak = "http://localhost:8080"
$env:ASPNETCORE_ENVIRONMENT = "Production"
.\Innovation.Web.exe
```

## Environment Variables (.env)

| Variable | Description | Default |
|----------|-------------|---------|
| `DOMAIN` | Web app hostname | *(required)* |
| `AUTH_DOMAIN` | Keycloak hostname | *(required)* |
| `WEB_IMAGE` | Web app container image | `awaleed0011/innovation-web:latest` |
| `WEB_PORT` | Web app internal port | `8080` |
| `POSTGRES_PASSWORD` | PostgreSQL password | *(auto-generated)* |
| `REDIS_PASSWORD` | Redis password | *(auto-generated)* |
| `KEYCLOAK_PASSWORD` | Keycloak admin password | *(auto-generated)* |
| `LDAP_ADMIN_PASSWORD` | OpenLDAP admin password | *(auto-generated)* |

## Important Notes

- **Keycloak runs in production mode** (`start`) behind Caddy's TLS termination.
- **Keycloak uses PostgreSQL** (`keycloakdb` database, created via init script) — not embedded H2.
- **Database migrations** run automatically on first startup.
- **Keycloak realm** is imported only on first startup. Update realm config via the admin console afterwards.
- **LDAP seed data** is loaded only on first OpenLDAP startup.
- **Passwords** are auto-generated by `publish.py`. The LDAP password is also injected into the Keycloak realm JSON.
- **Volumes** persist data across restarts: `postgres-data`, `redis-data`, `ldap-data`, `ldap-config`, `caddy-data`.
- **Network isolation**: only Caddy is exposed externally. All other services communicate on an internal Docker network.
